Introduction

Welcome to the documentation for the Package Explorer page. This document will guide you through the various features and functionalities available on this page.

Configuration

In this section, you can configure various settings:

  • Minimum Recursive Layer File Size: This means that if you were to take all of the packages in any given generated layer and calculat the summation of their file sizes and the file sizes of their recursive references, it will be no less than the value specified here. You would increase this value if you want to reduce the number of layers being generated (which increases the efficiency of the build).
  • Maximum Recursive Layer File Size: This means that if you were to take all of the packages in any given generated layer and calculat the summation of their file sizes and the file sizes of their recursive references, it will be no greater than the value specified here. You would increase this value if you want to reduce the number of layers being generated by allowing a given layer to introduce more new data to the container (which increases the efficiency of the build but can mean but also means that more of a given container will have to be rebuilt if a package changes).

Package List Usage

Here's how you can generate a Package List on the Package Explorer page:

  1. Step 1: Choose whether you want to display the top or botton N packages by entropy, dependency weight, file size or reverse dependency weight. Where N is the specified value of packages to be displayed.
  2. Step 2: Use the Add button and Remove button to specify any contraints for the packages to do with entropy, dependency weight, file size and reverse dependency weight.
  3. Step 3: Choose by which of the metrics the top / bottom N values should be sorted by, and which project the packages should be from.
  4. Step 4: Press the Generate Package List button.
  5. Step 5: Click on a package to see more information about it such as its Dependency Weight or to explore it's dependency tree.

Store Path Comparison Usage

Here's how you can use the Store Path Comparison Tool on the Package Explorer page:

  1. Step 1: Ensure that you have already generated a package list (please refer to the Package List Usage section of the documentation).
  2. Step 2: Click on a package to add either one of its store paths or one of the store paths of one of its recursive dependencies as either the base or the comparsion branch in the tool. Note that you can explore the dependency tree of a different store path of a package by first clicking on the store path, second selecting a different one and finally pressing the Go button.
  3. Step 3: Once you have both a base branch and a comparison branch, you can compare the branches by pressing the Compare Branches button.
